# 비상품 보증서 UI 스펙 (Figma 디자인 참고용)

상품 보증서 공유 페이지(https://buyer.collection.naver.com/collection/share/3Aq1pY0U) 구조를 참고하여, 비상품 보증서 타입별로 동일한 UI 패턴을 적용할 때 사용하는 스펙이다.

---

## 1. 상품 보증서 페이지 UI 구조 (참조)

| 영역 | 구성 요소 | 비고 |
|------|-----------|------|
| **상단** | 컬렉션 제목 (예: "○○님의 컬렉션") | H1 수준 |
| | 본문 바로가기 링크 | 선택 |
| | "○○님으로부터 공유됨" 문구 | 보조 텍스트 |
| **카드** | 서비스/브랜드명 (예: 아디다스피트니스) | 카드 상단, 브랜드 뱃지 느낌 |
| | 보증서/상품명 한 줄 (예: 아디다스 풀핑거 러닝 장갑 …) | 카드 메인 타이틀 |
| | 키워드/태그 (축구, 헬스, 등산, 자전거, 겨울 등) | 선택, 태그 형태 |
| | CTA 1: "상품 보러 가기" (외부 링크) | 주 액션 |
| | CTA 2: "네이버 컬렉션 소개" (가이드 링크) | 보조 액션 |
| **공통** | BRAND GUARANTEED / Brand-Certified Product / NAVER 등 브랜딩 | 상품의 경우; 비상품은 "비상품 보증서" 등 구분 뱃지 |

## 2. 비상품 보증서 타입별 카드 스펙

동일한 카드 레이아웃을 유지하고, **서비스명·제목·설명·CTA**만 타입별로 치환한다.

| 항목 | 내용 |
|------|------|
| **서비스/브랜드** | 해피빈 |
| **카드 제목** | 815 마라톤 기부증서 (또는 "해비타트×션 815런 기부 참여") |
| **설명/태그** | 기부 참여, 기부금 사용 인증, 연말정산 세액공제 연계 |

## 3. Figma에서 그릴 때 권장 구조

- **1프레임 = 1페이지**: 상단(컬렉션 제목 + 공유 문구) + 카드 리스트.
- **카드 컴포넌트 1종**: 변형(Variant)으로 타입만 바꾼다.
  - Props: 서비스명, 카드제목, 설명/태그, CTA1 문구, CTA2 문구, 뱃지 텍스트.
- **타입별 예시 프레임**: 해피빈 기부, 치지직 1호 팬, 치지직 구독, 815런 완주, MY골프, 교육 수료 각 1장씩 배치하면 팀 소개용으로 활용하기 좋다.
